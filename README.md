# 🧬 四进制基因编程演化系统

**DNA赋能版 - 真实生物基因驱动的程序演化实验平台**

基于DNA四进制编码的人工生命演化实验系统。支持导入真实物种基因序列，通过遗传算法实现程序自我演化。包含64指令虚拟机（生物学启发）和16指令虚拟机（演化使用）。

## 🧬 64指令虚拟机详解

### 设计原则

基于生物学遗传密码原理：
- **3位四进制密码子**：每个指令用3位四进制编码（000-333）
- **64种指令**：4³ = 64，完全利用编码空间
- **简并性**：关键指令有多个等价密码子（仿照DNA密码子简并性）
- **科学分组**：4大类 × 16指令/类

### 完整指令集 (0-63)

#### 组0: 控制流 (0-15)
| 十进制 | 四进制 | 指令 | 功能 | 简并 |
|--------|--------|------|------|------|
| 0-3    | 000-003 | NOP/HALT×3 | 空操作/停机 | HALT有3个备份 |
| 4-7    | 010-013 | START×2/RESET/DEBUG | 起始/重置/调试 | START有2个备份 |
| 8-11   | 020-023 | OUTPUT×2/INPUT/PRINT | I/O操作 | OUTPUT有2个备份 |
| 12-15  | 030-033 | CALL/RET/SYSCALL/TRAP | 函数调用/系统调用 | - |

#### 组1: 常量入栈 (16-31)
| 十进制 | 四进制 | 指令 | 功能 |
|--------|--------|------|------|
| 16-19  | 100-103 | PUSH_0/1/2/5 | 入栈常用小数 |
| 20-23  | 110-113 | PUSH_10/42/100/-1 | 入栈常用大数 |
| 24-27  | 120-123 | PUSH_PI/E/TRUE/FALSE | 入栈特殊常量 |
| 28-31  | 130-133 | PUSH_IMM/IMM2/NULL/RAND | 立即数/空值/随机数 |

#### 组2: 栈操作 (32-47)
| 十进制 | 四进制 | 指令 | 功能 |
|--------|--------|------|------|
| 32-35  | 200-203 | DUP/DUP2/SWAP/SWAP3 | 复制/交换栈顶 |
| 36-39  | 210-213 | POP/POP2/ROT3/ROT4 | 弹出/旋转栈顶 |
| 40-43  | 220-223 | LOAD/STORE/LOAD_FAST/STORE_FAST | 内存/局部变量 |
| 44-47  | 230-233 | BUILD_LIST/TUPLE/DICT/UNPACK | 数据结构操作 |

#### 组3: 算术运算 (48-63)
| 十进制 | 四进制 | 指令 | 功能 | 简并 |
|--------|--------|------|------|------|
| 48-51  | 300-303 | ADD×2/SUB×2 | 加减法 | 各有2个备份 |
| 52-55  | 310-313 | MUL×2/DIV×2 | 乘除法 | 各有2个备份 |
| 56-59  | 320-323 | MOD/POW/SQRT/ABS | 模/幂/根/绝对值 | - |
| 60-63  | 330-333 | NEG/INC/DEC/FLOOR | 取负/自增/自减/取整 | - |

### 简并性设计（容错机制）

| 功能 | 主密码子 | 简并密码子 | 容错率 | 生物学对应 |
|------|----------|------------|--------|------------|
| HALT | 001 | 002, 003 | 3x | 终止密码子(UAA/UAG/UGA) |
| START| 010 | 011 | 2x | 起始密码子(AUG) |
| OUTPUT| 020 | 021 | 2x | - |
| ADD  | 300 | 301 | 2x | 亮氨酸(6个密码子) |

### 编程示例

#### 示例1: Hello World (输出42)
```
DNA序列: 010 111 020 001
翻译:
  010 → START    # 程序开始
  111 → PUSH_42  # 入栈: 42
  020 → OUTPUT   # 输出: 42
  001 → HALT     # 停机
结果: [42]
```

#### 示例2: 计算2+3
```
DNA序列: 010 102 102 101 300 300 020 001
翻译:
  010 → START
  102 → PUSH_2   # [2]
  102 → PUSH_2   # [2, 2]
  101 → PUSH_1   # [2, 2, 1]
  300 → ADD      # [2, 3]
  300 → ADD      # [5]
  020 → OUTPUT   # 输出5
  001 → HALT
结果: [5]
```

## ❓ 常见问题

### Q1：64指令VM和16指令VM有什么区别？

- **16指令VM**：用于演化系统，简单高效，便于遗传算法搜索
- **64指令VM**：独立模块，基于生物学遗传密码原理设计，功能更丰富，支持简并编码

### Q2：为什么适应度增长缓慢？

- 检查基因长度（推荐120+）
- 降低变异率（推荐0.01）
- 增加种群大小（推荐100-200）

### Q3：为什么适应度始终为0？（适应度荒漠）

**原因**：
- ❌ DNA质量太差，转换后的指令完全无效
- ❌ 任务太难，初始种群距离解空间太远

**解决方案**：
1. 🎯 降低任务难度（先用 `target_number` 验证）
2. 🔄 更换DNA（使用已知有效的胰岛素基因）
3. 📈 混合初始化（50%DNA + 50%随机）
4. ⚡ 早期终止（100代内无进展立即停止）

#### 📝 详细修复说明

**1. 适应度修复**
- **问题**：v2.0日志显示适应度289.9770，远超理论上限1.0
- **原因**：`genetic_evolution/fitness.py` 中使用 `1000.0 / (1.0 + distance)` 缩放
- **修复**：改为 `1.0 / (1.0 + distance)`，范围现在正确为 0.0-1.0
- **验证**：运行1000代实验，最佳适应度稳定在1.0

**2. 指令解码BUG修复**
- **问题**：OUTPUT(31)、HALT(33)指令永远无法执行
- **原因**：编码规则 `opcode = val // 4 * 10` 只能生成0/10/20/30，而判断用 `opcode == 31`
- **修复**：改用operand子分发，`opcode == 30` 时根据operand(0-3)分别对应INPUT/OUTPUT/NOP/HALT
- **测试**：所有输出和停机指令现在正常工作

## 📚 技术背景
